## Narrator Script

If you train four days a week, meditate every morning, or review your finances every Monday, you shouldn't have to create that task manually every time. Recurring tasks automate the generation of repetitive scheduled work so your planner stays current without extra effort.

---

### What Recurring Tasks Are

A recurring task is a pattern, not a single entry. You define the task once — what it is, when it happens, how often — and the system generates individual task instances according to that pattern.

Each generated instance is a real task in your planner: it has a date, time, activity name, and all the other fields. You check it off just like any manually created task. The difference is that the next instance generates automatically.

---

### Creating a Recurring Task

Go to `/dashboard/planner` and click **+ Recurring Task** (or the recurring task button in the toolbar).

The recurring task form opens.

---

### The Recurring Task Form

**Milestone** — Select which milestone this recurring task belongs to. Every recurring task must be linked to a milestone, just like regular tasks. Start typing the milestone name and select from the dropdown.

**Activity** — What you're doing. Same as a regular task activity name.

**Description** (optional) — Your standard session protocol, a template you'll follow each time, or notes about the recurring activity.

**Tag** — Which category.

**Priority** — P1, P2, or P3.

**Time** — The time of day for this task (HH:MM format).

**Recurrence pattern** — This is the key field. Choose from:

---

### Recurrence Patterns

**Daily** — Creates a task every single day. Good for: meditation, journaling, daily review, stretching routines.

**Weekly** — Creates a task on specific days of the week. You select which days: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday (any combination). This is the most common pattern. Good for: training schedules (Mon/Wed/Fri/Sat), weekly meetings (Tuesday mornings), or habits on specific days.

**Biweekly** — Creates a task every two weeks (every 14 days). Good for: fortnightly check-ins, bi-weekly meal prep sessions, long-run training (when combined with a weekly shorter run).

**Monthly** — Creates a task on a specific day of the month (e.g., the 1st, the 15th, the last day). Good for: monthly financial reviews, monthly goal check-ins, cycle-based habits.

**Custom** — Creates a task every N days of your choice. If you want something every 3 days, or every 5 days, custom handles it. Good for: progressive training loads, recovery cycles, unconventional schedules.

---

### Setting an End Date (Optional)

Most recurring patterns can have an optional end date. If you're setting up a training block that runs through a specific milestone date, set the end date to match. After that date, no more instances will be generated.

If you don't set an end date, the pattern continues indefinitely until you pause or delete the recurring task.

---

### Pausing or Removing a Recurring Task

If a recurring task is no longer relevant — you're injured, the goal has changed, the habit has become automatic enough that you don't need to track it — you can deactivate the recurring pattern. This stops future instances from generating. Existing instances (already generated) are unaffected.

---

### Recurring vs. Manual Tasks

Use recurring tasks for:
- Regular training sessions or workouts
- Daily habits (meditation, journaling, stretching)
- Weekly processes (review, meal prep, planning session)
- Recurring commitments (classes, appointments, scheduled work blocks)

Use manual tasks for:
- One-off events (a specific race, a presentation, a launch day)
- Sessions with non-standard content that changes each time
- Tasks you need fine-grained control over

Many users use both: recurring tasks for the scaffolding of weekly routines, plus manual tasks for specific events and projects. The combination means you always have a realistic schedule in the planner without spending a lot of time on setup.

## Key Takeaways

- Recurring tasks generate individual task instances automatically based on a pattern
- Must be linked to a milestone — each instance is a real task in your planner
- Five patterns: Daily, Weekly (select days), Biweekly, Monthly (specific day), Custom (every N days)
- Optional end date — use it to align with a milestone's target date
- Deactivate a pattern to stop future generation; past instances are unaffected
- Best for regular habits and training schedules; use manual tasks for one-off events
