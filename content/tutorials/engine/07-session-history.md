## Narrator Script

Every focus session you run is stored in Session History. This is your complete log of focused work — searchable, filterable, and editable. You can review what you worked on, correct mistakes in logged data, and build a clear picture of how you're actually spending your focused time.

---

### Navigating to Session History

From the Engine dashboard, click **Session History** in the Engine navigation. You'll see a list of all your focus sessions, most recent first.

---

### What the History List Shows

Each row in the session history shows:
- **Date and time** — when the session started
- **Activity** — what you worked on
- **Duration** — actual focused time logged
- **Mode** — Simple or Pomodoro indicator
- **Quality rating** — the 1–5 rating you gave the session
- **Milestone / Task** — linked plan items (if set)
- **Revenue / Cost** — financial data (if set — see below)

---

### Filtering and Searching

At the top of the history list, you can filter by:

**Date range** — View sessions from a specific period. Useful for reviewing a particular week or month.

**Activity search** — Type any keyword to filter sessions by activity name. If you always prefix writing sessions with "Write:", searching "Write:" shows only writing sessions.

**Quality filter** — Filter to show only sessions above or below a certain quality rating. Useful for reviewing your best sessions ("what was I doing when I was focused?") or your worst ("when do I struggle most?").

**Milestone filter** — Filter to sessions linked to a specific milestone.

**Mode filter** — Simple only, Pomodoro only, or both.

---

### Editing a Session

Sessions don't always get logged correctly. Maybe you forgot to stop the timer and it ran for 3 hours, or you set the wrong activity name, or you want to add a milestone link after the fact.

Click **Edit** on any session row. The session edit form opens with all fields editable:
- Activity name
- Duration (manual override)
- Quality rating
- Milestone and task link
- Start and end notes
- Revenue and cost (optional financial fields)

Make your changes and click **Save**. The updated session data propagates to your stats and weekly review totals.

---

### Adding Revenue and Cost to a Session

Focus sessions have optional financial fields: **Revenue** and **Cost**. These are separate from the financial fields on tasks in the Planner — they're for logging the financial output of a specific work session.

Common uses:
- **Revenue**: You did a billable client call (log the invoice value), shipped a product feature that closed a deal, or completed a freelance deliverable
- **Cost**: You paid for a tool, software, or resource used in this session

These session-level financials aggregate into the Engine analytics and appear in the financial summary for the weekly review period.

---

### Duplicating a Session

If you ran a session that was nearly identical to a previous one (same activity, same duration, same mode), you can duplicate a past session to pre-fill the session setup form.

Click **Duplicate** on any history row. The session setup form opens with all fields from that session pre-filled. Adjust the date, modify any details, and click Start (or Save if you're logging it retroactively without running the timer).

This is useful for recurring work that doesn't follow a fixed schedule — less structured than a template, more convenient than setting up from scratch.

---

### Deleting a Session

Click **Delete** on any session row. A confirmation prompt appears. Confirm to permanently remove the session from your history and stats.

Use this for accidental sessions (you forgot to stop the timer), test entries, or truly erroneous logs. Don't use it to remove sessions that just had poor quality ratings — low-quality sessions are valid data.

---

### Retroactive Logging

You can log a session retroactively without running the timer. From Session History, click **+ Log Session**. Fill in all fields including the date (set to any past date). This is useful if you did focused work but forgot to run the timer.

The retroactively logged session is identical to a timer-run session in every way — it shows in stats, analytics, and the weekly review.

## Key Takeaways

- Session history shows all focus sessions: date, activity, duration, mode, quality rating, milestone/task
- Filter by date range, activity keyword, quality rating, milestone, or mode
- Edit any session to correct duration, quality, activity, notes, or add financial data
- Revenue/Cost fields on sessions are for session-level financial output (separate from task financials in Planner)
- Duplicate a session to pre-fill the setup form from a past session
- Log sessions retroactively with "+ Log Session" — identical to timer-run sessions in all stats
- Delete is permanent — use for errors, not for low-quality sessions (those are valid data)
